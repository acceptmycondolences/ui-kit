# SearchBar

```tsx
import { SearchBar } from 'ui-kit/core'
```

Компонент `SearchBar` — расширяемая строка поиска с анимированным раскрытием, поддержкой управляемого и неуправляемого состояния.

---

## Использование

```tsx
<SearchBar onClear={handleClear} onSearch={handleSearch} placeholder="Placeholder" />
```

---

## Состояния

| Состояние      | Описание                                                                 |
| :------------- | :----------------------------------------------------------------------- |
| `aria-invalid` | Фон и текст переключаются на деструктивную цветовую схему                |
| `closed`       | Отображается только кнопка-иконка; поле свернуто и недоступно для фокуса |
| `disabled`     | Pointer events отключены; поле и кнопки отображаются с прозрачностью 40% |
| `filled`       | Появляется кнопка очистки; нажатие на иконку поиска вызывает `onSearch`  |
| `open`         | Поле разворачивается с анимацией; кнопка переключается в активный режим  |

---

## Режимы управления

### Неуправляемый

Состояние открытия управляется внутри компонента. `defaultOpen` задает начальное значение.

```tsx
<SearchBar defaultOpen onClear={handleClear} onSearch={handleSearch} placeholder="Placeholder" />
```

### Управляемый

Состояние открытия передается снаружи через `open` и `onOpenChange`.

```tsx
<SearchBar onClear={handleClear} onOpenChange={setIsOpen} onSearch={handleSearch} open={isOpen} placeholder="Placeholder" />
```

### С начальным значением

`initialValue` устанавливает значение извне и принудительно держит поле открытым. При изменении пропа значение синхронизируется.

```tsx
<SearchBar initialValue={query} onClear={handleClear} onSearch={setQuery} placeholder="Placeholder" />
```

## Взаимодействие

| Действие                                | Результат                                                     |
| :-------------------------------------- | :------------------------------------------------------------ |
| Клик по иконке (закрыто)                | Открывает поле, фокусирует после анимации                     |
| Клик по иконке (открыто, поле пустое)   | Закрывает поле, возвращает фокус на кнопку                    |
| Клик по иконке (открыто, есть значение) | Вызывает `onSearch(value)`                                    |
| `Enter` (есть непустое значение)        | Вызывает `onSearch(value)`                                    |
| `Escape`                                | Закрывает поле, очищает значение, вызывает `onClear`          |
| Клик по кнопке очистки                  | Очищает значение, вызывает `onClear`, возвращает фокус в поле |

---

## Свойства

Компонент принимает все нативные атрибуты `<input>` (кроме `defaultValue`, `onChange`, `type`, `value`), а также:

| Свойство       | Тип                       | По умолчанию | Описание                                                                                   |
| :------------- | :------------------------ | :----------: | :----------------------------------------------------------------------------------------- |
| `defaultOpen`  | `boolean`                 |   `false`    | Начальное состояние открытия для неуправляемого режима. Игнорируется при наличии `open`    |
| `initialValue` | `string`                  |      —       | Значение, синхронизируемое снаружи. При наличии принудительно держит поле открытым         |
| `onClear`      | `() => void`              |      —       | Вызывается при очистке поля кнопкой или при закрытии с непустым значением                  |
| `onOpenChange` | `(open: boolean) => void` |      —       | Вызывается при изменении состояния открытия. Обязателен для управляемого режима            |
| `onSearch`     | `(value: string) => void` |      —       | Вызывается при подтверждении запроса через `Enter` или клик по иконке при заполненном поле |
| `open`         | `boolean`                 |      —       | Управляемое состояние открытия. При передаче компонент переходит в контролируемый режим    |

## Доступность

| Механизм        | Описание                                                                                 |
| :-------------- | :--------------------------------------------------------------------------------------- |
| `aria-controls` | Кнопка-переключатель ссылается на `id` инпута                                            |
| `aria-expanded` | Отражает текущее состояние открытия поиска                                               |
| `aria-invalid`  | Семантическое обозначение ошибки для скринридеров                                        |
| `id` / `useId`  | `id` инпута формируется через `useId` или принимается через проп                         |
| `sr-only`       | Кнопки содержат скрытые текстовые метки: `Open`, `Close`, `Search`, `Clear`              |
| `tabIndex`      | Инпут и кнопка очистки исключаются из tab-порядка в закрытом состоянии (`tabIndex={-1}`) |

## CSS API

| Токен                                | Описание                                          |
| :----------------------------------- | :------------------------------------------------ |
| `--accent-foreground-disabled`       | Цвет иконки кнопки очистки в состоянии `disabled` |
| `--constructive`                     | Цвет каретки; фон кнопки в открытом состоянии     |
| `--constructive-disabled`            | Фон кнопки в состоянии `disabled`                 |
| `--constructive-foreground-disabled` | Цвет иконки кнопки в состоянии `disabled`         |
| `--destructive`                      | Фон поля в состоянии `aria-invalid`               |
| `--destructive-foreground`           | Текст и каретка в состоянии `aria-invalid`        |
| `--destructive-foreground/40`        | Кольцо (ring) в состоянии `aria-invalid`          |
| `--muted`                            | Фон поля                                          |
| `--muted-foreground`                 | Цвет placeholder и иконки в закрытом состоянии    |
