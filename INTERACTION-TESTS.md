# Матрица интерактивных тестов Storybook

Матрица определяет, где `play`-тесты в Storybook дают реальную защиту поведения, а где достаточно визуальной story и a11y-проверки Storybook.

## Принципы

- Используйте `play` только для пользовательского поведения после рендера: ввод, клик, keyboard flow, открытие/закрытие overlay, выбор значения, callback side effects.
- Проверяйте поведение через доступные роли, labels и текст: `within(canvasElement)`, `userEvent`, `expect`, `waitFor`, `fn` из `storybook/test`.
- Не тестируйте CSS-варианты, размеры, цвета и Tailwind-классы через interaction tests. Для них достаточно stories и Chromatic.
- Не дублируйте поведение Radix UI полностью. Проверяйте только публичный контракт обертки: что trigger открывает контент, disabled блокирует действие, выбранное значение меняется, close/action работает.
- Если story использует portal, сначала взаимодействуйте с элементом в canvas, затем ищите portaled content через `within(document.body)` или `screen`, когда элемент находится вне canvas.
- Для story с `play` не отключайте `parameters.interactions`; для purely visual stories можно оставлять interactions disabled.

## Порядок документа

- Матрица покрытия перечисляет core-компоненты в порядке public API из `README.md`, затем addon-компоненты.
- Частичные списки компонентов сохраняют тот же относительный порядок, даже если сгруппированы по приоритету внедрения или типу поведения.

## Порядок внедрения

1. Сначала покрыть `Button`, `Checkbox`, `Field`, `Input`, `SearchBar`, `Select`, `Tabs`.
2. Затем покрыть overlay-компоненты: `AlertDialog`, `Dialog`, `Drawer`, `DropdownMenu`, `Popover`, `Sheet`.
3. Затем покрыть compound/search/form flows: `Command`, `InputOTP`, `Toaster`, `Form`.
4. Не добавлять `play` в `Alert`, `Badge`, `LoadingDots`, `Progress`, `Table`, пока у них нет пользовательского поведения или нового публичного a11y-контракта.

## Критерии готовности

- В каждом файле stories с `play` включены interactions и импортируются только нужные utilities из `storybook/test`.
- Callback-и в stories проверяются через `fn()`, а не через actions addon.
- Сценарии используют доступные селекторы, а не CSS-классы или test ids, кроме случаев без доступной альтернативы.
- После добавления очередной группы тестов запускается `pnpm test-storybook`; для крупных изменений дополнительно `pnpm lint` и `pnpm check`.
